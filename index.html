Background

From integrals to learned operators

The standard approach to −u″ = f is to discretise the domain and solve a linear system Au = f — once per forcing function. The Green's function G = A⁻¹ precomputes this inverse so any f is solved with a single integral: u(x) = ∫ G(x,y) f(y) dy. For problems where G has no closed form, a neural operator learns the same mapping f(·) ↦ u(·) directly from data.

The four demos below trace this arc in order — direct solver, analytic kernel, integral operator, learned operator. New to PDEs? Start with the background reading. Want a broader survey of classical solvers? See methods for solving PDEs.

Finite differences

Discretise −u″ = f to get a tridiagonal system Au = f, solved in O(N) via the Thomas algorithm.

🎯

Green's function

G = A⁻¹. Each column is the response to a point source — precompute once, apply to any f instantly.

∫

Integral operator

u(x) = ∫ G(x,y) f(y) dy — the solution as a weighted sum of impulse responses.

🧠

Neural operator

DeepONet learns branch(f) · trunk(x) ≈ u(x), implicitly recovering G from examples alone.
