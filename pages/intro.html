Background reading

What is a PDE?
From $\mathcal{L}_a\, u = f$ to neural operators

A self-contained introduction to partial differential equations as operator equations, and the question that motivates learning solution maps from data.

§ 1

Equations that relate a function to its own rates of change

An ordinary differential equation (ODE) asks: find a function $u(x)$ whose derivatives satisfy some prescribed relation. The simplest non-trivial example is $u' = u$, whose solution is the exponential $u(x) = e^x$. A partial differential equation (PDE) generalises this to functions of several variables — but even in one spatial dimension the essential character is the same: the unknown $u$ appears alongside its own derivatives.

Throughout these demos we work with a single spatial variable $x \in [0, 1]$ and the equation

The governing equation
$$-\frac{d}{dx}\!\left(a(x)\,\frac{du}{dx}\right) \;=\; f(x), \qquad u(0) = u(1) = 0.$$

Here $f(x)$ is a given forcing function — something we impose on the system — and $u(x)$ is the unknown response we want to find. The function $a(x) > 0$ is a coefficient that describes a physical property of the medium: in heat conduction it is thermal conductivity; in groundwater flow (Darcy flow) it is permeability. Everything interesting about the problem lives in the interplay between $a$, $f$, and $u$.

§ 2

Thinking in operators: $\mathcal{L}_a\, u = f$

It is useful to give the left-hand side a name. Define the differential operator

The parameterised operator
$$\mathcal{L}_a \;:=\; -\frac{d}{dx}\!\left(a(x)\,\frac{d}{dx}\right).$$

With this notation our equation becomes simply $\mathcal{L}_a\, u = f$. Think of $\mathcal{L}_a$ as a machine that takes a function $u$ and returns another function. It is linear: $\mathcal{L}_a(\alpha u + \beta v) = \alpha\,\mathcal{L}_a u + \beta\,\mathcal{L}_a v$ for any constants $\alpha, \beta$. Linearity is what makes the whole theory of Green's functions possible.

Expanding the product rule, $\mathcal{L}_a u = -a(x)\,u''(x) - a'(x)\,u'(x)$. When the coefficient is constant, $a(x) = 1$, the second term vanishes and we recover the familiar Poisson equation:

Special case: Poisson equation ($a \equiv 1$)
$$-u''(x) = f(x), \qquad u(0) = u(1) = 0.$$

This is the equation solved in all four interactive demos. It is the simplest non-trivial member of the family $\{\mathcal{L}_a\}_{a > 0}$, and it admits a closed-form Green's function — which is what makes it such a clean starting point.

§ 3

The forward problem: given $a$ and $f$, find $u$

Solving $\mathcal{L}_a\, u = f$ for $u$ — given $a$ and $f$ — is called the forward problem. In the constant-coefficient case, the standard numerical approach is to discretise the domain on a uniform grid of $N+1$ interior points with spacing $h = 1/(N+1)$. The second derivative is approximated by the centred finite-difference stencil:

Finite-difference approximation
$$-u''(x_i) \;\approx\; \frac{-u_{i-1} + 2u_i - u_{i+1}}{h^2} \;=\; f_i.$$

Writing this for every interior point simultaneously produces a linear system $\mathbf{A}\mathbf{u} = \mathbf{f}$, where $\mathbf{A}$ is the familiar tridiagonal matrix with $2/h^2$ on the main diagonal and $-1/h^2$ on the two adjacent diagonals. Because of this special structure, the system can be solved in $O(N)$ operations via the Thomas algorithm (forward elimination followed by back substitution).

§ 4

The Green's function: precomputing the inverse

Solving the system afresh for every $f$ is wasteful when the operator $\mathcal{L}_a$ — and therefore $\mathbf{A}$ — stays fixed. The Green's function $G_a(x, y)$ is defined as the solution to

Defining equation for $G_a$
$$\mathcal{L}_a\, G_a(x, y) \;=\; \delta(x - y), \qquad G_a(0,y) = G_a(1,y) = 0,$$

where $\delta(x - y)$ is a Dirac delta — a unit point source located at $y$. In matrix language, $G_a$ encodes the columns of $\mathbf{A}^{-1}$: each column is the solution when $f$ is a unit spike at a single grid point.

Once $G_a$ is known, any forcing $f$ can be handled without another solve. By linearity and superposition, the solution is simply the integral of $G_a$ against $f$:

Solution via the Green's function
$$u(x) \;=\; \int_0^1 G_a(x,\, y)\; f(y)\; dy.$$

For the Poisson equation ($a \equiv 1$) the Green's function is known analytically:

Closed form for $a \equiv 1$
$$G_1(x, y) \;=\; \min(x, y)\cdot\bigl(1 - \max(x, y)\bigr).$$

This is a piecewise-linear tent function — symmetric in $x$ and $y$, peaking at $(x, y) = (x_0, x_0)$ along the diagonal, and vanishing at the boundaries. Its symmetry reflects the self-adjointness of $\mathcal{L}_1$.

§ 5

Why parameterise by $a$?

The Poisson equation is clean but atypical. In practice the coefficient $a(x)$ varies across the domain — it might represent spatially heterogeneous soil permeability in a groundwater model, or locally varying thermal conductivity in a heat-transfer simulation. Every distinct $a$ gives a different operator $\mathcal{L}_a$, and therefore a different Green's function $G_a$.

This is the crux of the difficulty. Even if computing $G_a$ for one fixed $a$ is feasible, doing it for many different coefficient fields — as arises in uncertainty quantification, design optimisation, or inverse problems — becomes prohibitively expensive. What we really want is not a single $G_a$ but the full solution operator:

The solution operator
$$\mathcal{S}: a \;\longmapsto\; \bigl(f \mapsto u\bigr),$$

a map that takes a coefficient field and returns the corresponding solver. In the constant-$a$ setting we have been using, $\mathcal{S}$ reduces to the map $f \mapsto u = \int G_1(\cdot, y)\,f(y)\,dy$. In the general case, $\mathcal{S}$ is a map between infinite-dimensional function spaces.

§ 6

Approximating $\mathcal{S}$ with a neural operator

A neural operator is a neural network architecture designed to approximate maps between function spaces — objects like $\mathcal{S}$. The key insight is that such a map must be discretisation-invariant: if we refine the grid from $N = 32$ to $N = 128$ points, the learned operator should still apply, without retraining.

The simplest neural operator is DeepONet (Lu et al., 2021), which represents the output as

DeepONet approximation
$$\hat{u}(x) \;=\; \underbrace{\mathbf{b}(f)}_{\text{branch net}} \cdot \underbrace{\mathbf{t}(x)}_{\text{trunk net}} + \beta,$$

where the branch net encodes the input function $f$ (evaluated at fixed sensor locations) into a vector, and the trunk net encodes the query point $x$. Their dot product approximates the integral $\int G_a(x, y)\,f(y)\,dy$ — the network is implicitly learning the kernel.

Once trained on pairs $(f_i, u_i)$ generated by the true PDE solver, the network evaluates in microseconds regardless of grid resolution — making it practical for the many-query settings where classical solvers become expensive.
